MODELS="ornstein supergemma trevorjs"

usage() {

  echo "Estado actual:"

Hay un problema importante que encontré en la comunidad. Hay reportes de que el mmproj del Gemma 3 12B abliterated no funciona correctamente con llama.cpp + Open WebUI — el modelo recibe la imagen pero responde como si no la viera. El mmproj existe pero la integración tiene issues conocidos con este stack específico.

Además hay otro punto crítico: Gemma 3 no es multimodal de la misma forma que Gemma 4. El mmproj de Gemma 3 fue creado por bartowski como experimento, no es oficial de Google.

Mi recomendación honesta para tu caso:

Mantén supergemma-vision para escenas generales — ya está validado y funciona perfectamente con Open WebUI.

Para el pipeline de generación de prompts gore/assets 3D, el modelo de visión no necesita ser uncensored por sí solo. El flujo correcto es:

supergemma-vision analiza imagen de referencia
        ↓
descripción técnica estructurada (sin contenido explícito)
        ↓
SuperGemma/TrevorJS recibe esa descripción
y genera el prompt gore completo para ComfyUI
La censura solo importa en el paso de generación del prompt, no en el análisis visual. SuperGemma y TrevorJS ya hacen ese trabajo sin censura. ¿Probamos ese flujo completo antes de buscar otro modelo de visión?
